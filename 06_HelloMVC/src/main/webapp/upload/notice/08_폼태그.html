프론트에서 작성한 다수의 데이터를 서버에 전송할때 사용하는 태그
form태그는 데이터를 받을 수 있는 input태그를 자석태그로 사용
from태그는 속성으로 action, method를 필수적으로 설정한다
action : 요청을 보낼 주소값을 설정
method: 데이터를 전송하는 형식을 설정(default- get)

method속성
GET: 데이터를 upl주소에 넣어서 전달
간단한 조회에 사용, 게시글조회 *전송되는 데ㅣㅇ터ㅔㄹ
POST: 데이터를 request body에 넣어서 전달하는 방식
전송되는 데이터에 제한 이없다

get방식



input에 작성된 데이터는 주소의 ?뒤에 key=value형식으로 전송된다.
예시) http://서버주소:포트번호/서비스맵핑주소?name속성값=value속성값&name속성값=value속성값...

입력형식에 대해 알아보기

사용자에게 데이터를 입력받을때는 input태그를 이용한다.
input태그의 type속성에 따라 입력받는 UI가 달라진다.
일반적으로 input태그는 form태그의 자식태그로 사용함
type에 따라 value값을 고정으로 지정해줘야하는 input태그도 있다.
input태그는 서버에 key=value형식으로 전달됨
from태그와 같이 사용한 input태그는 반드시 name속성을 작성해줘야한다

type:text * 기본문자를 입력받을때 사용 가장 많이 사용

ype:password * 보안필요한 문자를 입력받을때 사용

사용자에게 출력되지않는 input태그

type:hidden

클라이언트에게 보여주지않지만 내부적으로 서버에서 필요한 데이터를 전송할때 사용 반드시 input내부에 name, value속성을 설정한다

button태그를 이용하는 방법
* 이벤트와 연결해서 사용 -> javascript로 구현한다 form태그안에 button태그를 기본으로 사용하면 이 버튼은 submit과 같은 기능을 함

제출과 취소하는 기능

submit: form태그에 설정된 action주소에 요청을 보내는 타입 * 버튼으로 표시
reset: form태그 내부에 있는 input태그에 작성된 값을 모두 초기화하는 기능

다중선택, 한개만 선택하는 태그

다중선택: checkbox -> 체크표시가 가능한 네모사각형을 출력

한개만 선택: radio -> 체크표시가 가능한 동그란모형을 출력

1. 두 타입 모두 다수의 태그로 생성했을때 name값을 동일하게 맞춰줘야함
2. 두 타입 모두 정해진값에 대해 value속성을 설정해줘야함
3. 어떤 내용을 표시하는 문구로 출력해줘야함
